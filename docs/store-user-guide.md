# Store User Guide
## Azee - Donut Shop Management System

### 1. Overview

Store users are responsible for managing daily operations at individual store locations. Your role focuses on delivery confirmation, waste reporting, inventory tracking, and maintaining communication with the production team to ensure optimal store performance.

**Key Responsibilities**:
- Confirm daily deliveries from production
- Report waste and damaged products
- Track store inventory and sales
- Monitor store performance metrics
- Communicate with production team
- Ensure quality standards at store level

### 2. Access & Permissions

#### 2.1 Store-Specific Access
- **Store Dashboard**: View your assigned store's data and metrics
- **Delivery Management**: Confirm and track deliveries
- **Waste Reporting**: Report damaged, unsold, or expired products
- **Inventory Tracking**: Monitor stock levels and product availability
- **Performance Analytics**: View store-specific statistics and trends
- **Communication Tools**: Interact with production and admin teams

#### 2.2 Restricted Access
- **Other Stores**: Cannot view data from other store locations
- **Production Planning**: Cannot create or modify production plans
- **User Management**: Cannot create or modify user accounts
- **System Configuration**: Cannot change global settings
- **Cost Data**: Limited access to sensitive cost information

### 3. Store Dashboard

#### 3.1 Dashboard Overview

**Key Metrics Displayed**:
- Today's delivery status

#### 3.2 Quick Actions

### 4. Delivery Management

#### 4.1 Daily Delivery Process
**Navigation**: Dashboard → Deliveries → Today's Deliveries

**Delivery Confirmation Steps**:
1. **Review Delivery List**: Check scheduled deliveries for the day
2. **Physical Inspection**: 
   - Verify quantities match delivery sheet
   - Check product quality and condition
   - Note any discrepancies or damages
3. **System Confirmation**:
   - Confirm received quantities
   - Report any shortages or overages
   - Note delivery time and condition
4. **Quality Assessment**: Rate overall delivery quality

**Delivery Status Types**:
- **Pending**: Delivery scheduled but not yet received
- **Partial**: Some items received, others pending
- **Complete**: All items received and confirmed
- **Issues**: Problems requiring attention

#### 4.2 Delivery Discrepancy Handling
**Common Discrepancies**:
- **Quantity Variances**: More or fewer items than ordered
- **Quality Issues**: Damaged or defective products
- **Missing Items**: Ordered items not delivered
- **Wrong Products**: Incorrect varieties or forms delivered

**Resolution Process**:
1. Document the discrepancy with photos if needed
2. Report immediately in the system
3. Contact production team for urgent issues
4. Follow up on resolution status

#### 4.3 Delivery Schedule Management
**Schedule Information**:
- Regular delivery times and days
- Special delivery arrangements
- Holiday schedule modifications
- Emergency delivery requests

### 5. Waste Reporting & Management

#### 5.1 Waste Categories
**Navigation**: Dashboard → Waste → Report Waste

**Waste Types**:
- **Delivery Damage**: Items damaged during transport
- **Production Defects**: Quality issues from production
- **Unsold Products**: Items not sold by end of day
- **Expired Items**: Products past sell-by date
- **Customer Returns**: Items returned by customers

#### 5.2 Waste Reporting Process
**Daily Waste Reporting**:
1. **End of Day Count**: Count unsold and damaged items
2. **Categorize Waste**: Classify by waste type and reason
3. **System Entry**:
   - Select product variety and form
   - Enter quantity and waste reason
   - Add notes for context
   - Submit report

**Best Practices**:
- Report waste daily for accuracy
- Be specific about waste reasons
- Take photos for quality issues
- Communicate patterns to production

#### 5.3 Waste Reduction Strategies
**Store-Level Actions**:
- **Display Optimization**: Arrange products attractively
- **Temperature Control**: Maintain proper storage conditions
- **Rotation Management**: First-in, first-out inventory rotation
- **Customer Education**: Promote day-old discounts
- **Feedback Loop**: Report demand patterns to production

#### 6.3 Sales Tracking

### 10. Daily Workflows

#### 10.1 Opening Routine (6:00 AM - 8:00 AM)
**Pre-Opening Tasks**:
1. Check delivery notifications in system
2. Prepare for incoming deliveries
3. Review yesterday's performance
4. Set up display areas
5. Check equipment and supplies

**Delivery Reception**:
1. Receive and inspect deliveries
2. Confirm quantities and quality
3. Update system with delivery confirmation
4. Arrange products in display cases
5. Report any issues immediately

#### 10.2 Operating Hours (8:00 AM - 6:00 PM)
**Throughout the Day**:
- Monitor inventory levels
- Maintain product displays
- Serve customers and collect feedback
- Track sales patterns
- Handle any quality issues

**Mid-Day Tasks**:
- Stock level assessment
- Display refreshing
- Customer feedback review
- Special order management

#### 10.3 Closing Routine (6:00 PM - 7:00 PM)
**End of Day Process**:
1. Count remaining inventory
2. Report waste and unsold items
3. Clean and secure displays
4. Complete daily reports
5. Prepare for next day's deliveries

### 13. Troubleshooting

#### 13.1 Common Issues
**System Access Problems**:
1. Verify login credentials
2. Check internet connection
3. Clear browser cache
4. Contact technical support

**Data Entry Issues**:
1. Verify required fields are completed
2. Check data format requirements
3. Save work frequently
4. Report persistent problems

**Delivery Discrepancies**:
1. Document with photos
2. Report immediately
3. Follow up on resolution
4. Maintain communication with production

#### 13.2 Emergency Procedures
**Urgent Quality Issues**:
1. Remove affected products immediately
2. Report to production team urgently
3. Document with photos and details
4. Follow up on investigation results

**System Outages**:
1. Use backup paper forms
2. Document all activities
3. Enter data when system is restored
4. Report outage to technical support
